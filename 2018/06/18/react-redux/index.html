react-redux

xwb
2018-06-18

react-redux的学习，主要关注两点 Provider和connect；
Provider的作用是将rudux创建的store传递到内部的其他组件。让内部组件可以享有这个store并提供对state的更新。
connect有四个参数，一般只会用到前两个。
export default connect(mapStateToProps,mapDispatchToProps)(MainPage);
mapStateToProps和mapDispatchToProps都为函数，返回的一定是个对象
mapStateToProps里有两个参数，第一个state是一开始的时候Provider里传过来的,这里面有所有你的数据，第二个ownProps也是父级那传过来的。
mapDispatchToProps代表的也是函数 (dispatch)=>{
   // console.log(actionCreators)
   return bindActionCreators(actionCreators, dispatch)
}
这个作用是，将所有的action结合起来，存放到this.props中。这样调用action的方法的时候，就不需要结构dispatch了。只需结构具体方法就可以了。
MainPage 具体的组件指的是，返回出的是哪一个组件，并且前面的信息数据，给哪个组件使用。
